01

エグゼクティブサマリー

familyai.jp は日本語で唯一の「家族×AI」ブランドになれる可能性を持つ。 号数表記（VOL.01 二〇二六年六月）や「⁂」アステリズム装飾で、 テックディレクトリではなく手帖・雑誌の情緒を確立している。 TechCrunch ではなく暮らしの手帖——このポジショニングは競合不在のブルーオーシャンだ。

技術面では Next.js 14 App Router + Drizzle ORM + Neon PostgreSQL の構成で、 GINインデックス戦略・ISR最適化・CSRF対策・レート制限が実装済み。 コードベースには日本語で詳細なコメントが残され、一人開発の文脈では資産価値が高い。

しかし約束と実態のギャップが最大の弱点である。 公称「100+AI活用事例」に対して実記事数は3本。 AIツールも教育カテゴリの2つのみで、掲げる4カテゴリのうち3つが空。 このギャップを埋めなければ、どんなにデザインが良くても来訪者の信用を失う。

最重要所見

familyai.jp の課題は「何かが壊れている」ではなく「まだ十分に作られていない」である。 デザインもコードも丁寧に作られているからこそ、コンテンツさえ追いつけば一気に化けるポテンシャルがある。 逆に、このままの状態が続けば、せっかくの世界観と技術投資が来訪者の失望によって無駄になる。

04

最大の課題 — 解決必須の3点

この3つを解決しない限り、成長軌道に乗るのは難しい。

ISSUE № 01 · 信用毀損

コンテンツ量が公称と大幅に乖離している

トップページに「100+AI活用事例」と掲げる一方、実際の記事は3本。これは単なる「まだ少ない」ではなく、来訪者の期待と現実の不一致——サイトの信用そのものを毀損するリスク。初回来訪者が記事一覧で「全3件」を見たときの落胆は、どんなにデザインが良くても回復できない。すぐに表記を実数ベースに修正し、記事を最短で10本まで増やす必要がある。

ISSUE № 02 · 戦略ギャップ

価値提案が広すぎて実態と合っていない

「家族の学びと暮らしをAIで応援」は4カテゴリを掲げるが、実際に提供できているのは「学習・教育」だけ。残り3カテゴリのコンテンツはゼロ。フォーカスを狭めて教育特化とするか、カテゴリを埋める努力をするか——戦略判断が必要な局面。どちらを選ぶにせよ、現状の「掲げるが提供できていない」状態が最も悪い。

ISSUE № 03 · 需要検証不足

「このサービスがなくなったら困る」ユーザーがまだ見えない

無料でツールを提供し、親が子どもに使わせる導線はあるが、リピート使用・課金意欲・口コミ紹介といった真の需要シグナルがまだ観測できていない。ユーザー行動データ（リテンション、セッション時間、ブックマーク率）を取得し、「なくては困る」と言うユーザーの声を集める段階にある。

05

改善ロードマップ — 4フェーズ計画

優先度順。即実行可能なものから中長期投資まで。

Phase 1 今週中 · 即実行

信用ギャップを塞ぐ

最優先。公称と実態の乖離はサイトの信頼性を直接損なう。

  • トップページの「100+AI活用事例」表記を現在の実数（3件）＋「毎週更新」に修正
  • 下書き2本（ai-homework-helper.md, english-learning-voice-ai.md）を今週中に公開
  • 404ページに検索窓＋カテゴリリンク＋人気記事3件を追加（離脱防止）
  • ヘッダーにグローバル検索を追加（実装コスト低・UX向上大）
Phase 2 1〜2週間

コンテンツの厚みを作る

量とカテゴリバランスを整え、来訪者が「また来たい」と思う状態に。

  • 記事10本を目標に執筆。空カテゴリ（家事・暮らし / 仕事・効率化 / 創作・表現）に最低1本ずつ
  • VOAツール：中級・上級の最初の1モジュールだけ実装して「準備中」率を下げる
  • 3D図鑑：新モデル1〜2点を追加（生物・物理カテゴリに1点ずつ）
  • トップページに「今週のピックアップ」セクション追加（更新頻度を可視化）
Phase 3 1〜2ヶ月

ユーザー導線とエンゲージメント

獲得した来訪者を定着させ、行動データを取得し始める。

  • 初回来訪者向けオンボーディング：「まず何をするか」を3ステップで提示
  • 記事↔ツールの相互リンクを実装（回遊率向上）
  • Vercel Speed Insights / カスタムイベントでユーザー行動分析を開始
  • X（@familyaijp）での週2回以上の定期投稿（記事更新通知＋AI活用Tips）
Phase 4 3〜6ヶ月

収益化と差別化の加速

ユーザーデータに基づいて課金仮説を検証し、持続可能なモデルに。

  • 課金仮説を1つ選んでMVP検証（VOAプレミアム / 3Dモデル追加課金 / 記事有料化 のいずれか）
  • ユーザー投稿型のAI活用事例コミュニティ（UGCで記事供給を加速）
  • メールマガジン / LINE公式でリテンションチャネルを確立
  • Schema.org 構造化データ（Article, WebApplication）で検索エンジン最適化
06

技術的クイックウィン — コードベースの低コスト改善

コードベース精査で見つかった、実装コストが低くインパクトの大きい改善点4件。

PRIORITY · 今すぐ

HTMLページのセキュリティ境界を明示

lib/html-page-auth.ts でパスワード保護HTMLの認証を実装しているが、Vercel Blobから配信されるHTMLコンテンツのXSSリスク評価が不十分。iframe sandbox属性で隔離するか、サニタイズレイヤーを追加。

PRIORITY · 1週間以内

未使用テーブルのデッドライン設定

user_animations テーブルは「Phase 2まで残置」と明記されているが、スキーマファイル内に長大なコメントで存在し続けている。2026 Q3までに着手しなければ削除するデッドラインを設定。

PRIORITY · 1週間以内

3DブックマークのAPI・UI接続

user_3d_bookmarks テーブル + リポジトリ層は実装済みだが、APIエンドポイントとUIの⭐ボタンが未実装（Codex Q1-5/Q2-6指摘事項）。残り30%の作業で機能が完成する。

PRIORITY · 1ヶ月以内

記事タグの動的管理に移行

現在タグは各記事のfrontmatterに手動記述。記事数が増えると表記揺れ（#ChatGPT / #chatgpt / #Chat-GPT）が発生しフィルターが破綻する。タグ正規化レイヤーまたは管理画面でのタグ選択式入力を導入。

最終所見

最も重要なこと

familyai.jp の課題は「何かが壊れている」ではなく「まだ十分に作られていない」です。 これは悪いニュースではありません。デザインもコードも丁寧に作られているからこそ、 コンテンツさえ追いつけば一気に化けるポテンシャルがあります。 逆に、このままコンテンツ不足の状態が続けば、せっかくの世界観と技術投資が 来訪者の失望によって無駄になります。

優先すべきは Phase 1（信用ギャップを塞ぐ）—— これは1週間で完了でき、かつ最もダメージの大きい問題を解決する。 次に Phase 2（コンテンツの厚み）で記事10本・ツール拡充により 「また来たい」サイトに変える。 ユーザーが定着し始めたら Phase 3〜4 で収益化と成長に着手する。
