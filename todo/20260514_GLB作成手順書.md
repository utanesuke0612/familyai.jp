`/tools/ai-kyoshitsu` にアップロードする **GLB ファイル**（3D モデル）を、本プロジェクトの仕様に**確実に合格する形で**作るための公式手順書。サーバ側 zod 検証・Blob upload-token のサイズ/MIME 制限・hotspot エディタのクリック判定すべてを満たすことを保証する。

## 🔒 0. 合格基準（必須要件）— これを全部満たせば必ず通る

| # | 項目 | 必須 | 上限・規約 | 違反時の挙動 |
|---|---|---|---|---|
| F1 | 拡張子 | ✅ | `.glb` 固定（小文字） | upload-token が `許可されていない拡張子です` で拒否 |
| F2 | MIME | ✅ | `model/gltf-binary` または `application/octet-stream` | Blob PUT が `Content-Type mismatch` で 400 |
| F3 | ファイルサイズ | ✅ | **30 MB** 以下（推奨 1〜2 MB／上限 5 MB でモバイル UX 確保） | Blob PUT が `maximumSizeInBytes` で 400 |
| F4 | アップロード先 URL | ✅ | `/3d-models/...` / `/api/...` / `*.public.blob.vercel-storage.com/...` のいずれか | `createTutor3dModelSchema` の `assetUrlSchema` で 400 |
| F5 | ファイル名（Blob 上） | ✅ | `3d-models/{slug}-{hash8}.glb` 形式（ハッシュ 8 文字でキャッシュバスト） | キャッシュ事故・差し替え時の旧 URL 残留 |
| F6 | 三角面数 | ⬜ | 推奨 < 50,000 ／ 上限 < 100,000 | 低スペック端末で 30fps 切る・`detect-gpu` フォールバック発動 |
| F7 | テクスチャ解像度 | ⬜ | 推奨 1024×1024 ／ 上限 2048×2048 | DL 時間増・モバイル遅延 |
| F8 | メッシュ/マテリアル名 | ⭐ | ASCII 英数字＋アンダースコア・パーツの英語名（例: `Earth`、`Heart_Material`） | hotspot のクリック判定（material 名一致）が無効化され「位置近似のみ」にフォールバック |
| F9 | 軸・スケール | ✅ | Y-up（glTF 標準）／ Y 軸高さを **1m 前後**に正規化 | model-viewer のデフォルトズームで枠外 or 米粒サイズ |
| F10 | アニメ長さ | ⬜ | 推奨 5〜10 秒ループ ／ 上限 30 秒 | 注意散漫・教育用途に不適 |
| F11 | hotspot 数 | ✅ | **最大 30 個**（`hotspotArraySchema.max(30)`） | DB POST が 400 |
| F12 | USDZ（iOS AR 用・任意） | ⬜ | 同じく 30 MB 以下／拡張子 `.usdz`／MIME `model/vnd.usdz+zip` | 無い場合 iOS Quick Look が使えないだけ・Android Scene Viewer は GLB で OK |

> ⭐ = **強く推奨**（守らないと一見動くが UX が壊れる項目）

### Step 2 — Blender で調整（DL ファイルの場合は必須・AI 生成も微調整推奨）

1. **インポート**: `File > Import > glTF 2.0`
2. **スケール正規化**: Y 軸高さを **1m 前後**に（N キーで右パネル → Dimensions）
3. **不要オブジェクト削除**: 背景・カメラ・ライト（model-viewer 側で照明する）
4. **減面**（面数 > 100,000 の場合）: `Modifiers > Decimate > Ratio 0.3〜0.5`
5. ⭐ **メッシュ・マテリアル命名**（**最重要**）
   - Outliner で各オブジェクト選択 → Object Properties で名前変更
   - 例: `Earth` / `Mars` / `Heart_Atrium` / `Pendulum_Bob`
   - マテリアルも同じ命名（例: `Earth_Material`）
   - **命名規則**:
     - ✅ ASCII 英数字 + アンダースコア
     - ❌ スペース・日本語・記号・全角文字
   - **なぜ重要か**: クリック判定が 2 段階で動く
     - ① マテリアル名一致（精度◎ ／ 誤検知ゼロ）
     - ② 一致しなければクリック位置近似（精度△）
6. **テクスチャ最適化**（任意）: Image Editor で 2048 → 1024 に縮小
7. **エクスポート**: `File > Export > glTF 2.0 (.glb)`
   - Format: **glTF Binary (.glb)** ✅
   - Geometry → **Apply Modifiers** ✅
   - Materials → **Export** ✅
   - Animation → **Export** ✅（あれば）
   - **+Y Up** ✅（標準）

> 💡 hotspot の `meshName` に `Earth` を指定すれば、Blender で `Earth` という名のオブジェクトをクリックした時に必ず命中する。

### Step 3 — Draco 圧縮（5〜10MB → 0.5〜2MB）

```bash
cd scripts/3d-tools

# 単体
./compress.sh output/solar-system.glb final/
# → final/solar-system.glb （Draco 圧縮済み）

# 一括
./compress.sh raw-models/ final/
```

内部実装は `gltf-pipeline -i input.glb -o output.glb -d`（`-d` = Draco）。

**圧縮後サイズの確認**:
```bash
ls -lh final/solar-system.glb
# → 期待値: 0.5MB 〜 2MB
```

⚠️ **2MB を超えた場合**:
- Blender で減面率を上げる（Decimate Ratio 0.3 → 0.2）
- テクスチャを 1024 → 512 に縮小
- 不要な PBR チャンネル削除（emissive 等）

### Step 5 — Hotspot 座標採取

```bash
# preview.html と同じディレクトリの hotspot-picker.html を開く
```

1. GLB ファイルを選択（ドラッグ or 選択）
2. 3D 上で **注釈したい場所をクリック** → hotspot ピンが追加
3. 各 hotspot に入力:
   - `partName`: パーツ表示名（日本語可・例「地球」）
   - `defaultExplanation`: タップ時の即時表示文（〜400 文字）
   - `promptHint`: AI 深掘り用ヒント（〜800 文字・system prompt に埋め込まれる）
   - `meshName`（任意）: Blender で付けた英名（例 `Earth`）
4. 「JSON ダウンロード」で `{slug}.json` を保存

**hotspot JSON フォーマット**（生成される形式）:
```json
[
  {
    "id":                 "h1",
    "partName":           "地球",
    "position":           [0.0, 0.0, 0.0],
    "normal":             [0, 1, 0],
    "defaultExplanation": "私たちが住む水の惑星。",
    "promptHint":         "地球の構造・大気・自転について。",
    "meshName":           "Earth"
  }
]
```

**制約**:
- 最大 **30 個** / モデル（zod で強制）
- `partName` ≤ 80 文字 / `defaultExplanation` ≤ 400 / `promptHint` ≤ 800
- `id` は 1〜40 文字（英数字推奨）

### Step 8 — 本番アップロード（管理者画面経由）

1. ブラウザで `/admin/3d-models/new` を開く（要 `lijun.kawasaki@gmail.com` ログイン）
2. メタ情報を入力:
   - `slug`: 小文字英数字＋ハイフンのみ（例 `solar-system`・1〜120 文字）
   - `title`: 表示タイトル（〜200 文字）
   - `description`: 説明文（〜2000 文字）
   - `subject`: `biology` / `chemistry` / `earth-space` / `physics`
   - `grade`: `elem-low` / `elem-high` / `middle`
3. **GLB ファイルを選択** → ブラウザが Vercel Blob に直接 PUT（5MB 超でも OK・`@vercel/blob/client` 経由）
4. USDZ ・サムネイル（任意）も同様にアップロード
5. **Hotspot JSON を貼り付け**（Step 5 で生成したファイルの中身）
6. `attribution` ・ `license` ・ `sourceUrl` を入力
7. `published: false` のままで保存 → プレビューで確認
8. 問題なければ `published: true` に切替

**アップロード命名規則**（自動付与・上書き不可）:
```
3d-models/{slug}-{hash8}.glb
3d-models/{slug}-{hash8}.usdz
3d-models/{slug}-{hash8}.{webp|png|jpg}
```

> ハッシュ 8 文字でキャッシュバスティング。差し替え時は新ハッシュなので CDN キャッシュ事故が起きない。

## 🆘 4. トラブルシューティング

### 「許可されていない拡張子です」が出る
→ ファイル名が `.glb` で終わっているか確認。大文字 `.GLB` は不可。

### Blob PUT が 400「Content-Type mismatch」
→ 一部ブラウザ・転送経路で MIME が `application/json` になる事故。`@vercel/blob/client` の `upload()` を使えば自動で正しく設定される（直接 fetch 禁止）。

### Blob PUT が 400「maximumSizeInBytes exceeded」
→ 30 MB を超えている。Draco 圧縮 → 減面 → テクスチャ縮小 の順で対応。

### hotspot クリックが反応しない
1. **meshName ベース不一致**: Blender でメッシュ・マテリアル名を ASCII で命名 (Step 2-⑤)。Codex P2 既知問題で material 名一致しない場合に位置近似 fallback が動かないことがある → meshName 一致 or 位置近似のどちらかで判定が通るように両方記録する。
2. **位置が GLB から大きく外れている**: `hotspot-picker.html` でモデル上に正しく刺さっているか再確認。

### model-viewer で何も表示されない
- DevTools コンソールで `Failed to load resource` を確認
- URL が `assetUrlSchema` を満たしているか確認（`/3d-models/...` または `*.public.blob.vercel-storage.com/...`）
- WebGL 対応ブラウザか（Chrome / Safari / Edge）

### 動くが「カクカク」or「重い」
- 三角面数を Blender Statistics で確認 → 50,000 以下に減らす
- テクスチャを 1024 以下に
- アニメをループ短縮（30 秒 → 10 秒）

### iOS AR が動かない
- USDZ が必要 → Reality Converter で生成（Mac）
- HTTPS でないと AR 不可（本番のみ・localhost は OK）

### Windows で USDZ 生成できない
- USDZ なしで公開（Android AR は GLB だけで動く）
- または [Vectary](https://www.vectary.com/) や [Aspose](https://products.aspose.app/3d/conversion/glb-to-usdz) でオンライン変換

## 📐 5. 早見表（コピペ用）

### サーバ側制約まとめ
```
拡張子:      .glb（必須）
MIME:        model/gltf-binary / application/octet-stream
最大サイズ:  30 MB
URL:         /3d-models/... | /api/... | *.public.blob.vercel-storage.com/...
ファイル名:  3d-models/{slug}-{hash8}.glb
hotspots:    最大 30 個
subject:     biology | chemistry | earth-space | physics
grade:       elem-low | elem-high | middle
slug:        ^[a-z0-9-]+$ ・1〜120 文字
```

### Blender Export 設定
```
Format:      glTF Binary (.glb)
Include:     Selected Objects（必要なものだけ）
Transform:   +Y Up ✅
Geometry:    Apply Modifiers ✅・UVs ✅・Normals ✅・Tangents（必要なら）
Material:    Export ✅
Animation:   Export ✅
Compression: Draco（gltf-pipeline 後段で必ず適用）
```

### 推奨ターゲット
```
ファイル: 1〜2 MB（圧縮後）
面数:     10,000〜30,000 三角形
テクスチャ: 1024×1024 ×（1〜3 枚）
アニメ:   5〜10 秒ループ
```
