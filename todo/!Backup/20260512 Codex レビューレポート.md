**良い点**
- Server Component でカタログ/詳細を取得しており、以前の巨大 Client Component より構造が良いです。
- `model-viewer` を Client Component に閉じ込めていて、SSRとの分離も妥当です。
- DB、shared型、zod schema、mapper、repository が分離されており、今後の管理画面/API拡張に向いています。
- GLB制作・圧縮・hotspot採取ツールまで用意されていて、運用フローを意識できています。

**改善ポイント**
| 優先度 | 改善案 |
|---|---|
| P1 | Hotspot判定のコメントと実装が少し不一致です。`materialFromPoint()` で material 名が取れたが hotspot と一致しない場合、現在は位置近似 fallback せず無反応になります。Tripo等の汎用 material 名モデルではクリックが効かない可能性があります。 |
| P1 | `/tools` では `adminOnly: true` で3D図鑑を隠していますが、`/tools/ai-kyoshitsu` 直URLには admin gate がありません。段階公開の意図ならページ側も `isAdmin()` で制御すべきです。 |
| P2 | `HotspotPanel` の SSE が delta ごとに `setMessages` しており、長文回答で再レンダリングが増えます。既存 `AIChatWidget` と同じ 50-100ms throttle に寄せるとよいです。 |
| P2 | カタログ文言に「DNA・振り子」とありますが、seed上は太陽系1件のみです。未提供モデルを出すと期待値ズレになるため、文言を現状に合わせるかモデルを追加してください。 |
| P2 | `user_3d_bookmarks` repository はありますが、UI/API/shared API は未接続です。お気に入り機能として使うなら `/api/user/3d-bookmarks` とマイページ導線が必要です。 |
| P2 | `/api/3d-models/[slug]` は詳細APIのみです。モバイル向けを想定するなら一覧API `/api/3d-models?subject=&grade=` も必要です。 |
| P3 | `glbUrl` は任意URLも許可されています。プライバシー/可用性を考えるなら `/3d-models/` または Vercel Blob の許可ドメインに制限した方が安全です。 |
| P3 | GLBのファイル名が `solar-system.glb` 固定なので、差し替え時にキャッシュ問題が出ます。`solar-system.v1.glb` やハッシュ付きファイル名を推奨します。 |
| P3 | ModelViewer に load error 用のUIがありません。GLB取得失敗時に「読み込めませんでした」「再試行」などを出すと運用しやすいです。 |

現状は「Phase 1: 管理者が用意した3Dモデルを図鑑として表示し、hotspotからAI解説できる」段階としてはかなり形になっています。次にやるなら、まず `Hotspot判定のfallback修正`、`直URLの公開制御確認`、`文言と実データの整合` を優先するのがよいです。

**総評**
3D図鑑への置換はかなり進んでいます。旧 `/api/generate-animation`、`/api/animations`、旧共有ページは現行ツリーから消えており、前回問題だった「生成HTML iframe sandbox」の主要な攻撃面はかなり削除されています。Server Component のカタログ/詳細、`model-viewer` の Client island、DB/repository/schema/mapper の分離も良いです。

一方で、「後续的可扩展性」では hotspot 判定の fallback 実装、公開制御の一貫性、ブックマーク機能の未接続が課題です。「不要なものが削除できているか」では、旧 `user_animations` 型/DB定義/AI教室定数、`.next` stale artifact、実データとずれた文言が残っています。

優先対応は、まず `adminOnly` の直URL制御、`tsc` が通る状態の復旧、hotspot fallback修正です。その後に旧アニメ関連の残骸を「削除するもの」と「Phase 2で残すもの」に分けて整理するのがよいです。
