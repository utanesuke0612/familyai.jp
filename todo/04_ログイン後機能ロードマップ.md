## TODO-L01: VOA単語帳のクラウド同期

**提案日**: 2026-04-24
**背景**: `/tools/voaenglish/vocab` は現在 localStorage ベースで動作している。
非ログインユーザーはそのまま使えるが、端末間の同期ができず、ブラウザデータ削除で消えてしまう。

### 現状（localStorage 版）

- ストレージ: ブラウザの localStorage（キー: `familyai:vocab:bookmarks`）
- 関連ファイル:
  - `lib/voaenglish/vocab-store.ts`（読み書き・React フック）
  - `components/article/AnnotatedWord.tsx`（☆ トグルUI）
  - `app/(site)/tools/voaenglish/vocab/page.tsx`（一覧ページ）
- データ書式: `VocabItem[]`（`id` / `word` / `meaning` / `pron` / `example` / `course` / `lesson` / `lessonTitle` / `addedAt`）
- 既存の出口: 画面右上の「📥 JSONで書き出す」で手動バックアップ可能

### 目指す姿（クラウド同期版）

- **非ログインユーザー**: 現状の localStorage 運用を継続（変更なし）
- **ログイン済みユーザー**: 
  - 書き込み／削除が DB に同期される
  - 別端末でも同じ単語帳が見える
  - 初回ログイン時、localStorage に既存データがあれば DB にマージする（重複は `id` で判定）

### 必要な実装

1. **DB スキーマ追加** (`lib/db/schema.ts`)
   ```ts
   export const userVocab = pgTable('user_vocab', {
     id:          uuid('id').defaultRandom().primaryKey(),
     userId:      uuid('user_id').notNull().references(() => users.id, { onDelete: 'cascade' }),
     vocabKey:    varchar('vocab_key', { length: 255 }).notNull(),   // course/lesson/word
     word:        varchar('word', { length: 255 }).notNull(),
     meaning:     text('meaning').notNull(),
     pron:        varchar('pron', { length: 100 }),
     example:     text('example'),
     course:      varchar('course', { length: 100 }),
     lesson:      varchar('lesson', { length: 100 }),
     lessonTitle: varchar('lesson_title', { length: 255 }),
     addedAt:     timestamp('added_at').defaultNow().notNull(),
   }, (t) => ({
     uniq: uniqueIndex('user_vocab_uniq').on(t.userId, t.vocabKey),
     idxUser: index('user_vocab_user_idx').on(t.userId),
   }));
   ```

2. **API**
   - `GET    /api/vocab`           — 自分の単語帳一覧
   - `POST   /api/vocab`           — 登録
   - `DELETE /api/vocab/:vocabKey` — 解除
   - `POST   /api/vocab/bulk`      — 一括マージ（初回ログイン時）
   - すべて `auth()` 必須、Zod バリデーション、レート制限（本 TODO 時点で既存 `getRateLimiter` 流用）

3. **`lib/voaenglish/vocab-store.ts` を拡張**
   - セッション状態（`useSession`）でログイン判定
   - 非ログイン: 現行 localStorage 経路（そのまま）
   - ログイン: fetch ベースで API 呼び出し、optimistic update + SWR ライクなキャッシュ
   - インターフェイス（`addVocab`/`removeVocab`/`useVocabList`）は同一を維持 → UI 側の変更不要

4. **初回マージ処理**
   - ログイン直後に一度だけ `localStorage` → `/api/vocab/bulk` へ POST
   - 成功後、`localStorage` はそのまま残す（ログアウト時のフォールバック兼バックアップ）
   - マージ戦略: `vocabKey` 一致なら DB 優先（`addedAt` が古いほう）、不一致は両方採用

5. **UI 追記**
   - `/tools/voaenglish/vocab` ページ上部に、
     - 非ログイン: 「ログインするとクラウドに保存され、別端末でも見られます →」CTA
     - ログイン済み: 「✅ クラウドに同期済み（最終同期: …）」
